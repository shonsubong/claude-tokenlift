# 위임 판단 상세 규칙

이 문서는 "무엇을 Ollama로 보내고 무엇을 Claude가 처리할지"의 의사결정 기준이다.
핵심 목표: **비싼 출력/입력 토큰을 로컬로 옮기되, 정확성과 판단이 중요한 부분은 Claude가 책임진다.**

## 의사결정 순서

1. **Claude 유지 신호가 있는가?** → 있으면 Claude. (보안/설계/근본원인/의사결정)
2. **위임 적합 신호 + 임계값 초과인가?** → 그러면 Ollama로 위임.
3. **애매하면** → `tokenlift route` 추천 참고, 기본은 Claude.

## Ollama로 위임 (✅ 적극 위임)

| 패턴 | 예시 요청 | 권장 명령 |
|---|---|---|
| 명세 명확한 신규 코드 | "이 인터페이스 구현체 작성" | `gen` |
| 보일러플레이트/스캐폴딩 | "CRUD 컨트롤러 5개 골격" | `gen` |
| 단위 테스트 | "이 모듈 테스트 작성" | `test` |
| 일괄 리팩터링 | "콜백을 async/await로", "함수 분리" | `refactor` |
| 언어/프레임워크 이식 | "이 파이썬을 Go로" | `translate` |
| 대용량 파일/로그 요약 | "이 5천 줄 로그에서 에러 원인" | `explain` |
| docstring/주석/문서 초안 | "공개 함수에 JSDoc" | `docs` |
| 정형적 변환 | "JSON 스키마 → 타입 정의" | `gen`/`translate` |

**위임 임계값(휴리스틱):** 생성 코드 30줄 이상, 처리 파일 300줄 이상, 동일 패턴 3개 파일 이상.
이 미만의 사소한 작업은 위임 왕복 비용(지연)이 절감액보다 클 수 있어 Claude가 바로 처리해도 된다.

## Claude가 처리 (❌ 위임 금지/주의)

| 패턴 | 예시 요청 | 이유 |
|---|---|---|
| 시스템/서비스 아키텍처 설계 | "전체 결제 시스템 설계" | 고난도 판단, 광범위 영향 |
| API/인터페이스/스키마 설계 | "공개 API 계약 정의" | 하위 호환·일관성 |
| 복잡한 디버깅·근본원인 분석 | "왜 간헐적으로 실패하나" | 다중 컴포넌트 추론 |
| 보안 민감 로직 | 인증/인가/암호화/결제 | 정확성·위험 |
| 전체 시스템 영향 리팩터링 | "도메인 모델 전면 개편" | 교차 의존성 |
| 요구사항 모호·트레이드오프 결정 | "어떤 방식이 나을까" | 맥락·책임 |
| **위임 결과의 최종 검토/통합** | (항상) | 품질 게이트 |

## 하이브리드 패턴 (권장)

대부분의 실제 작업은 "설계는 Claude, 생산은 Ollama, 검토는 Claude"로 나뉜다.

```
사용자: "알림 모듈 만들어줘 (이메일/SMS/푸시 채널)"

1. Claude  : 인터페이스·추상화·에러 전략 설계 (고난도, 짧은 출력)
2. Ollama  : 각 채널 구현체 3종 생성   → tokenlift gen ... (대량 출력, 위임)
3. Ollama  : 각 구현체 테스트 생성     → tokenlift test ... (대량 출력, 위임)
4. Claude  : 생성물 검토·보안 점검·통합 (짧은 판단)
```

이렇게 하면 가장 비싼 부분(2,3의 대량 코드 출력)이 로컬에서 처리된다.

## 모델 선택 요약 (task → model)

| task | 기본 모델 | 비고 |
|---|---|---|
| gen/edit/test/refactor/translate/explain | `qwen2.5-coder:14b` | 코드 주력 |
| review | `deepcoder:latest` | 리뷰 특화 |
| agent (멀티파일 에이전트형) | `devstral:24b` | 에이전트 코딩 |
| reason (추론 위임) | `deepseek-r1:14b` | `<think>` 자동 제거됨 |
| docs | `gemma3:12b` | 산문 |
| fast (간단/저지연) | `gemma3:4b` | 경량 |
| complete (FIM) | `qwen2.5-coder:1.5b-base` | 중간채우기 |

`-m` 플래그로 강제 지정 가능. 모델을 자주 바꾸면 재적재 비용이 크므로, 한 세션에선
가능한 한 `qwen2.5-coder:14b` 하나로 묶어 위임하는 것이 빠르다.

## 안티패턴 (하지 말 것)

- ❌ 보안/결제 코드를 위임하고 검토 없이 그대로 사용
- ❌ 5줄짜리 사소한 수정을 위임(왕복 지연 > 절감)
- ❌ 모호한 요구를 그대로 위임(로컬 모델이 엉뚱하게 해석)
- ❌ 매 호출마다 다른 모델 사용(재적재로 느려짐)
- ❌ Ollama 산출물을 Claude가 다시 전량 재생성(절감 효과 상쇄)
